# /notion-sync

Sync a company profile (or the full digest) to Notion using the Notion MCP.

---

## Usage

```
/notion-sync [company]       # Sync one company profile to Notion
/notion-sync --all           # Sync every company in companies/
/notion-sync --digest        # Sync the latest weekly digest page
```

---

## What this command does

### Step 1 — Load setup state

Read `context/notion-setup.md`.

- If it exists and contains a `tracker_database_id`, proceed to Step 3.
- If it does **not** exist (first run), go to Step 2.

---

### Step 2 — First-time setup (create Notion structure)

Only runs once. Creates the persistent Notion structure for this project.

#### 2a. Create a parent page

Use the Notion MCP (`notion-create-pages`) to create a top-level page titled **"Talent Intelligence"** in the user's workspace.

#### 2b. Create the Tracker database

Inside the parent page, use `notion-create-database` to create a database called **"Company Tracker"** with these properties:

| Property | Type | Notes |
|---|---|---|
| Company | title | Primary key |
| Stage | select | Options: Researched, Monitoring, Stale, Archived |
| Headcount | number | |
| HQ | rich_text | |
| Last Updated | date | |
| Hiring Signal | select | Options: 🔴 High, 🟡 Medium, 🟢 Low, ⚪ Unknown |
| Tech Stack | multi_select | |
| Profile Page | url | Link to the individual Notion page |
| Source File | rich_text | e.g. companies/stripe.md |

#### 2c. Save IDs to context

Write the following to `context/notion-setup.md`:

```markdown
# Notion Setup

> Auto-generated by /notion-sync. Do not edit manually.

## IDs

- **parent_page_id**: [id from step 2a]
- **tracker_database_id**: [id from step 2b]

## Status

First run completed: [today's date]
```

---

### Step 3 — Sync a company profile

Read the company file at `companies/[company-slug].md`.

Extract the following fields (use `---` not present or `❓` if missing):

- `company_name`
- `headcount` (number only, strip ~)
- `hq`
- `hiring_signal` (derive from recent job count: >20 open roles = High, 5–20 = Medium, <5 = Low)
- `tech_stack` (array of top 5 technologies)
- `stage` (default: Researched)
- `last_updated` (today's date)

#### 3a. Check for existing tracker row

Use `notion-query-data-sources` to search the Company Tracker database for a row where `Source File` = `companies/[slug].md`.

**If found:** update the existing row using `notion-update-page` with the extracted fields.

**If not found:** create a new row using `notion-create-pages` in the tracker database.

#### 3b. Create or update the company detail page

Check `context/notion-setup.md` for a `page_id` entry for this company.

**If no page exists yet:**

Use `notion-create-pages` to create a new page under the parent "Talent Intelligence" page.

Title: `[Company Name] — Intelligence Profile`

Page content (use markdown blocks):
```
# [Company Name]

> Synced from talent-intelligence/companies/[slug].md
> Last updated: [date]

---

[Full contents of the company markdown file, converted to Notion blocks]
```

Save the new page ID back to `context/notion-setup.md`:
```
- **[slug]_page_id**: [new page id]
```

**If page already exists:**

Use `notion-update-page` to overwrite the page content with the latest version of the company file.

#### 3c. Link tracker row → detail page

Update the `Profile Page` property of the tracker row with the URL of the detail page.

---

### Step 4 — Sync digest (--digest flag)

Read the most recent file in `reports/` matching `*-digest.md`.

Create or update a Notion page titled **"Weekly Digest — [date]"** under the parent "Talent Intelligence" page.

Post the full digest content as the page body.

Add a `📋 Digest` tag/label so digests are visually distinct.

---

### Step 5 — Sync all (--all flag)

List all files in `companies/`. Run Step 3 for each one, in alphabetical order.

Print a summary when done:
```
✅ Notion sync complete
   Companies synced: 7
   New pages created: 2
   Rows updated: 5
   Tracker DB: [notion url]
```

---

## Error handling

- If Notion MCP returns an error, print the error message clearly and do **not** crash silently.
- If a field can't be extracted from the markdown, use a safe default (`---`, `❓`, or `0`) rather than failing.
- If `context/notion-setup.md` is corrupted, offer to re-run setup (Step 2) and prompt the user to confirm before overwriting.

---

## Integration with other commands

This command is automatically called at the end of:

- `/research [company]` → runs `/notion-sync [company]`
- `/update [company]` → runs `/notion-sync [company]`
- `/update --stale` → runs `/notion-sync --all`
- `/digest` → runs `/notion-sync --digest`

To suppress Notion sync in any command, append `--no-notion` to the command call.
